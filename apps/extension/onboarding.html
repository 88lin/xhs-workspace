Browser Companion

Collect first. Shape later.

XHS Atelier Collector handles the browser side of the workflow. Capture Xiaohongshu material here, then move it into the desktop studio for organization, creation briefs, manuscripts, and RedClaw runs.

What this extension does Capture posts, clean them into exportable packages, and keep the raw material flowing into the desktop workspace.
What stays in desktop Long-term knowledge management, manuscript editing, archive history, and the creation pipeline.

Get oriented in under a minute

You do not need to configure everything at once. Start with local capture, then add AI analysis and the desktop bridge when ready.

1

Open the settings panel

Use the extension popup to review storage, export, and AI options. AI is optional, so local capture works even before you add any model provider.

2

Visit xiaohongshu.com

The extension attaches to supported Xiaohongshu pages and prepares the collection panel inside the browsing context.

3

Capture and review notes

Collect the notes you care about, optionally run AI analysis, and package the results as structured exports for later use.

4

Send material into XHS Atelier Desktop

Export to the bridge directory or save files locally, then import them inside the desktop app to continue with Library, Manuscripts, and RedClaw.

Read extension docs
Need help? Visit the issue tracker or the repository documentation. Captured data stays local by default unless you explicitly send content to your configured AI provider.
